## Interfaces Between RF System Controllers

## Introduction

SPEAR3 is designing a replacement for the certain components of the system that currently exists.  The new components will consist of a low-level radio frequency (LLRF) controller commercial system to control the amplitude and phase of the RF fields in the cavities, a new controller for the high voltage power supply (HVPS), and a new machine protection system (MPS) controller for RF components.  The high-power elements of the RF system, including the klystron, HVPS, and the cavities will remain as is.
We will need to interface the various components together.  The most important interfaces are those between the three components mentioned above.  During normal operation we want the LLRF system to be able to adjust the value of the HVPS output so that the klystron operates at its desired operating point.  This control occurs at a low update rate, on the order of one second, and can be handled using the standard software communications protocol of the EPICS-based control system.
However, we also need to enforce fast, real-time control for machine protection.  If the LLRF system detects a problem with the RF power, it should and will disable itself and the RF power by redundantly removing the RF input to the drive amplifier.  The HVPS outputs more power than the collector of the klystron can dissipate, so that if the drive power is turned off the HVPS must be turned off as well.
There are also elements within the SPEAR vacuum chamber that, in the event of an equipment malfunction, can be damaged by the power from the x-ray beam generated by the electrons in the SPEAR ring.  The SPEAR ring MPS therefore also needs to be able to dump the electron beam as quickly as possible.  It will do this by removing a permit to the LLRF so that the LLRF will disable itself and remove the RF input to the drive amplifier.
## Interconnections Between the System Components

We intend to design and build a chassis that will function as the interface between the LLRF, RF MPS, SPEAR MPS, and the HVPS controller.
### Interface Chassis

#### Input Permits (All signals give permits when active)

- LLRF permit for HVPS to turn on – optocoupler
- HVPS controller signal that HVPS is ready to be turned on – fiber optic receiver (Broadcom/Avago HFBR-2412)
- SPEAR PPS permit for RF system – 24 VDC input
- SPEAR orbit interlock permit for RF system – 24 VDC input
- RF MPS permit for HVPS – 24 VDC input
- Several spare inputs, perhaps one for 24 VDC and one for 5 VDC.
#### Output Permits (All signals will give permits when active)

- LLRF permit to enable LLRF output – optocoupler
- HVPS controller to enable phase control thyristors – fiber optic transmitter (Broadcom/Avago HFBR-1412)
- HVPS controller to inhibit output crowbar – fiber optic transmitter (Broadcom/Avago HFBR-1412)
#### Output Status

- Status signals to RF MPS system to show status of all input inhibit signals and the status of the three output permits – 24 VDC logic?
#### General Architecture

As stated above, all permits will be active high so that a break in the cable will fail safely.  All inputs will be optically isolated.  We will standardize on a standard Broadcom/Avago type optocoupler, such as HCPL-2400-000E.  These have a response time of .  They require a minimum input current drive of .  If we design for a  input with some safety margin, we can use a  resistor in series with the optocoupler input.  We may use jumpers for the outputs corresponding to the spare inputs when they are not in use.
The outputs of the input optical chips will feed a logical AND gate.  That gate will drive the HFBR-1412 transmitters to the HVPS controller.  Broadcom recommends a drive of  for each of these transmitters, with a recommended driver circuit in Figure 11 of its data sheet.  The current thinking is that the crowbar enable to the HVPS will always be enabled.  We will only remove the permit for the phase control thyristor trigger enable.
A fault of the RF MPS or the removal of the fiber optic status from the HVPS controller will remove the permit that is sent to the LLRF controller.
If the RF MPS system detects any system fault, it will send a software command, via the control system, to instruct the HVPS controller to turn off.  This will be a soft redundant signal for the removal of the phase control thyristor trigger enable.
